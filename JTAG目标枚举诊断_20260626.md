## 判读

Windows USB 层能看到一个 FTDI `VID_0403&PID_6014` 设备和 CP210x COM3，但 Vivado `hw_server` 没有看到任何 JTAG target。

这说明当前至少存在以下可能之一：

1. 开发板 JTAG 口、供电、复位、BOOT/JTAG 拨码或跳线状态不正确。
2. 当前接入的 USB 设备只是普通 FTDI 通道，未形成 Vivado 可识别的 JTAG cable。
3. FTDI/Digilent 驱动绑定不符合 Vivado 2023.1 当前 hw_server 的期望。
4. 开发板上电或 JTAG 链路异常，导致 USB 设备存在但链上没有可打开的 target。

## 继续硬件测试的门槛

在任何 2-lane AB/BA 通断测试、ILA capture、ELF 启动或 bitstream 烧写前，必须先运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_hw_target.ps1 -ComPort COM3 -JtagFrequencyHz 1000000
```

必须看到：

```text
HW_PREFLIGHT_RESULT PASS
HW_PREFLIGHT_ZYNQ ...
```

如果仍是：

```text
HW_PREFLIGHT_TARGET_COUNT 0
HW_PREFLIGHT_RESULT FAIL_NO_TARGET
```

则继续上板脚本没有工程意义，也不应驱动 TFDU 小板。

## 建议的人工检查顺序

1. 确认 AX7010 开发板主电源已打开，板上电源指示灯正常。
2. 确认 USB-JTAG 线接到开发板 JTAG/下载口，而不是只接串口或 USB-UART。
3. 确认 BOOT/JTAG 相关拨码、跳线、PS/PL 复位处于可 JTAG 调试状态。
4. 重新插拔 JTAG USB 线后，再运行 `tools/check_hw_target.ps1`。
5. 如仍失败，用管理员权限运行 Vivado cable/Digilent 驱动安装或在设备管理器中检查 `VID_0403&PID_6014` 的驱动绑定。
